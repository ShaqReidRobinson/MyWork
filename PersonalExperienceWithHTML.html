Week One - Learning about HTML 5 and the basics

Published on October 05, 2012

During the first week of Web Development, we began learning the basics tools used within Web Development such as Adobe Dreamweaver, HTML-Kit, Notepad++ etc.

We also began learning the ins and outs of HTML 5 along with the standards and how we will be applying it to our work, such as combining it with CSS in order to make a 'flashy' website.

I personally have very little to no experience with web development, so this was all very new to me.

Week Two - HTML Markup Basics

Published on October 12, 2012

Within this lecture, we started talking about various HTML terminology that we will be constantly using during our work.

We started off learning about "Elements" and "Attributes" and various rules that come with them, such as 'tags must always be closed' and 'elements must be in lowercase'.

Structure played a big part during the lecture as it is very important to learn how to structure your HTML wisely to avoid errors and mediorce presentation.

Week Three - HTML Section elements and hypertext.

Published on October 19, 2012

Within this lecture, we learnt about various HTML elements such as Div elements and section elements.

We applied this to our websites in order to make it more structured and tidy, thus making coding the website a lot less arduous.

With applying div/section elements to our work, we can easily style our website without the need to enter constant lines of descriptive CSS coding everytime we want to style something.

With a quick div element edit, we can easily apply the selected style of our choice to whatever

Week Four - Cascading Style Sheets (CSS)

Published on October 26, 2012

This week's lecture was focused on the amazing things you can do with CSS

We were also taught on how important it is to style a website accordingly, in order to present ourselves more professionally to the Internet. Having a website with dull text like Times New Roman and/or no colour whatsoever will imply to the people browsing the website that you are boring, lazy, and can't design properly or even design at all.

We were shown good examples of CSS being used successful to produce eye-catching webpages that had:

  • Decorative Borders
  • Opacity
  • Rounded Corners
  • Animation

Just to name a few.

Although we were given a large range of tools at our exposal from this lecture, we were warned to keep our website designs simple but creative, as "Less is More"

Week Five - CSS Box model and layout

Published on November 02, 2012

Today's lecture covered more CSS layouts, how to properly use them, and The CSS Box Model

We started off the lecture to more or less an introduction to what the CSS Box Model is, Another important way on how to structure our wesbite using it's components such as:

  • Margin
  • Padding
  • Border

Structures... STRUCTURES EVERYWHERE

From learning about this, we also took in account things such as Pixels Per Page and how it will fit on various devices such as Androids to Desktops

We also spoke briefly about adding colour to the box model and how we would go about doing that.

Week Seven - Writing and styling hypertext

Published on November 16, 2012

Today's lecture was about styling some hypertext. Simple. This involves styling things such as internet links, web text for search engine purposes, and various fonts.

We also touched on Search engine mechanics and how web users view each internet link accordingly whether it was eye-catching or not.

Week Eight - Images for the Web

Published on November 23, 2012

Images... Very crucial when coding your CSS. Especially when choosing from ranges of file types such as GIF, Vector, and other types of image effects. What is most important about using images on your website is positioning them corresponding to how your text is laid out

What I mean is that you wouldn't want an image blocking your text, so be careful when embedding images/videos into your website.

Week Eleven - Testing, Referencing and Evaluation

Published on December 14, 2012

I have now began testing the website thoroughly for any bugs I may have overlooked. To aid me with this, I'm constantly validating my HTML using the W3C Validator, with also helps lay out my code more neatly.

In conclusion, I consider my website to be above my expectations marginally as I felt that I could have made the CSS more stylish and eye-catching, but however, most of my ideas for such design required JavaScript which restricted my imagination as I'm not fluent in JavaScript yet.

Also, whilst positioning my website, I wasn't sure how I was going to position it. Whether to have things float left, right or center, or even have the webpage be viewed horizontally rather than a traditional vertical view. In the end, I managed to come to equal ground with it and positioned it well in my own opinion.

However, when embedding my video into my tutorial I didn't have the correct codecs for it to be viewed in Mozilla Firefox, and can only be viewed in Google Chrome. The advanced stylesheet for the tutorial is a bit temparamental in Google Chrome as the CSS will sometimes not work for unknown reasons or if the page has been refreshed on that stylesheet. This corrects itself when I click on the "Beginner's Tutorial" link in the navigation bar. The navigation bar may sometimes disappear when the CSS is not working, this also corrects itself from being refreshed. This problem doesn't occur in Firefox and works perfectly, disregarding the missing video codecs.

On the other hand, creating a website that runs fine and look presentable was the number one priority, as I did manage to fulfill that objective, which indeed is a good thing for only eleven weeks worth for lectures.
